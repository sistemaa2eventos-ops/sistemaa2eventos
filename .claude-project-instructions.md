## 📌 INSTRUÇÕES PERMANENTES

Quando trabalhar neste projeto, SEMPRE:

### 1. **Reconhecer o Contexto de Deployment**

Se o usuário mencionar qualquer um destes termos:
- "deploy", "subir", "lançar", "produção"
- "docker", "container", "nginx"
- "Supabase", "Cloudflare", "Hostinger"
- "atualizar código", "mudança no sistema"

**Ação:** Lembrar o usuário sobre `/deploy` skill ou oferecer protocolo padrão.

---

### 2. **Padrão de Deploy Obrigatório**

NUNCA fazer deploy sem:
- ✅ Verificar `.env` está correto
- ✅ Confirmar `git status` limpo
- ✅ Ter >5GB de espaço em disco
- ✅ Testar Supabase connectivity
- ✅ Usar `./deploy.sh full` ou `./deploy.sh quick`

Se usuário quiser fazer algo não-padrão, avisar:
```
⚠️ Isso não segue o protocolo padrão em CLAUDE.md
   Recomendo usar: ./deploy.sh full
   Ou consulte: /deploy para detalhes
```

---

### 3. **Localização de Arquivos**

Se usuário perguntar "onde está X?":
- Primeiro consulte: `SYSTEM_MAP.md`
- Se não encontrar, procure em `CLAUDE.md`
- Se ainda não encontrar, use `Glob` tool com padrões relevantes

---

### 4. **Verificações Automáticas**

Sempre que fizer mudança no backend ou frontend:
- [ ] Lembrar de fazer `docker-compose build --no-cache`
- [ ] Confirmar que `docker-compose ps` mostra "Up"
- [ ] Rodar `curl http://localhost:3001/health`
- [ ] Verificar logs: `docker logs a2-eventos-api --tail=20`

---

### 5. **Configurações Críticas**

Estes arquivos NUNCA devem ser modificados sem protocolo:
- `docker-compose.yml` ← Requer rebuild completo
- `.env` ← Requer reload dos containers
- `nginx.conf` ← Requer restart do nginx
- `CLAUDE.md` ← Requer comunicação com usuário

Se modificar, SEMPRE avisar:
```
⚠️ Mudança crítica detectada em [arquivo]
   Requer: [ação necessária]
   Tempo: [tempo estimado]
```

---

### 6. **Erros Conhecidos & Soluções**

Se encontrar estes erros, SEMPRE consulte seção no CLAUDE.md:

| Erro | Seção |
|------|-------|
| 502 Bad Gateway | Troubleshooting → Erro 4 |
| 408 Timeout | Troubleshooting → Erro 5 |
| CORS error | Troubleshooting → Erro 7 |
| Cannot connect Docker | Troubleshooting → Erro 1 |
| Port in use | Troubleshooting → Erro 2 |
| Supabase error | Troubleshooting → Erro 3 |
