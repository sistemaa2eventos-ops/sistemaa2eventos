🚀 DEPLOY COMMAND (Escolha um)
─────────────────────────────────────────────────────────────────────────────

  ./deploy.sh full       ← COMPLETO (20-30 min) - Usar sempre que possível
  ./deploy.sh quick      ← RÁPIDO (10-15 min) - Mudanças pequenas
  ./deploy.sh check      ← VERIFICAR (2 min) - Sem fazer mudanças

  ./deploy.sh logs       ← VER LOGS em tempo real

📋 BEFORE EVERY DEPLOY
─────────────────────────────────────────────────────────────────────────────

  1. git status          # Sem mudanças não-commitadas
  2. git fetch origin    # Buscar código remoto
  3. cat .env | head -5  # Verificar variáveis críticas
  4. df -h               # Ter >5GB livres
  5. ping painel.nzt.app.br  # Conectividade teste

✅ TESTES APÓS DEPLOY
─────────────────────────────────────────────────────────────────────────────

  L1: curl -s http://localhost:3001/health | jq .
      → Deve retornar: { "status": "ok" }

  L2: curl -s http://localhost:3000 | grep -q "DOCTYPE"
      → Se retorna, Frontend OK

  L3: curl -s -I https://painel.nzt.app.br
      → Deve ser 200 ou 301

  L4: docker-compose logs --tail=50 | grep -i "error"
      → Não deve ter erros críticos

🔴 SOS — ALGO DEU ERRADO
─────────────────────────────────────────────────────────────────────────────

  502 Bad Gateway?
    → docker logs a2-eventos-api --tail=20
    → curl http://localhost:3001/health

  Cannot connect to Docker?
    → sudo systemctl start docker

  Port already in use?
    → sudo lsof -i :80
    → sudo kill -9 [PID]

  Supabase error?
    → echo $SUPABASE_URL
    → curl -s -I $SUPABASE_URL

  ROLLBACK RÁPIDO:
    → git revert HEAD
    → git push origin master
    → ./deploy.sh quick
